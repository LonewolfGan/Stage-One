# PAGE RECHERCHE/RÉSULTATS — Carte + cards paginées

> S'appuie sur `tailwind.config.ts` déjà en place (noms sémantiques
> uniquement : `bg-canvas`, `text-ink`, `rounded-card`, etc.). N'invente
> aucune valeur en dehors de ce fichier — si une couleur/taille/radius
> manque, demande l'extension du config plutôt que d'écrire une valeur en
> dur.

## Référence structurelle (à adapter, pas à copier visuellement)

La structure suivante — carte interactive en haut, résultats en dessous —
est la bonne structure à reprendre :
- Carte plein cadre en haut de la zone de résultats, avec des markers
  personnalisés (icône salon plutôt qu'icône maison) qui changent de style
  selon le type de prestataire ou la disponibilité.
- Compteur de résultats ("X établissements trouvés") + ville, en haut à
  gauche, hors de la carte.
- Toggle de tri (équivalent "Sort By") en haut à droite.
- **Différence volontaire par rapport à la référence** : les cards de
  résultats ne flottent PAS par-dessus la carte. Elles sont dans une zone
  séparée juste en dessous, avec **pagination explicite** (numéros de page
  ou "Précédent/Suivant"), pas un scroll infini ni un chevauchement visuel
  avec la carte.

## Librairie de carte — Mapbox GL JS via react-map-gl

Mapbox est le bon choix ici : très bon free tier (largement suffisant pour
le développement et le lancement), et surtout un contrôle total sur le
style visuel de la carte via Mapbox Studio — ce qui permet de faire matcher
la carte avec le design system du projet (palette claire, pas le bleu/vert
saturé par défaut de Google Maps). Google Maps reste pertinent si on a
besoin de Street View ou d'une couverture de données plus large, mais ce
n'est pas notre cas.

```bash
npm install react-map-gl mapbox-gl
```

// .env.local

### Style de carte personnalisé

Dans Mapbox Studio (studio.mapbox.com), créer un style basé sur "Mapbox
Light" et ajuster :
- Couleur des routes principales → gris clair proche de `surface-3`
  (`#E9E7E2`)
- Couleur de l'eau → gris-bleu très clair, pas le bleu saturé par défaut
- Labels de villes/quartiers → couleur proche de `ink-secondary`
  (`#54534D`)
- Désactiver les POI par défaut de Mapbox (restaurants, commerces) pour ne
  garder que les markers de salons — sinon ça surcharge visuellement
- Récupérer l'URL du style personnalisé (`mapbox://styles/...`) et
  l'utiliser dans le composant.

Si la création du style custom dans Mapbox Studio n'est pas faisable dans
l'immédiat, utiliser le style `mapbox://styles/mapbox/light-v11` par défaut
en attendant — c'est le plus proche visuellement de notre palette claire.

## Structure du composant

```tsx
"use client";
import Map, { Marker, NavigationControl } from "react-map-gl";
import "mapbox-gl/dist/mapbox-gl.css";
import { useState } from "react";

interface SalonResult {
  id: string;
  name: string;
  city: string;
  latitude: number;
  longitude: number;
  rating: number;
  reviewCount: number;
  priceLevel: string; // "€" | "€€" | "€€€"
  nextSlot: string;
  imageUrl: string;
  badge?: string; // "Populaire" | undefined
}

function SearchResultsPage({ results, totalCount, city }: {
  results: SalonResult[];
  totalCount: number;
  city: string;
}) {
  const [selectedId, setSelectedId] = useState<string | null>(null);
  const [page, setPage] = useState(1);
  const perPage = 8;
  const totalPages = Math.ceil(totalCount / perPage);

  return (
    <div className="max-w-page mx-auto px-5 md:px-10 lg:px-20 py-8">
      {/* En-tête résultats */}
      <div className="flex items-start justify-between mb-6">
        <div>
          <h1 className="text-heading-l text-ink">
            {totalCount} établissements trouvés
          </h1>
          <p className="text-body-s text-ink-tertiary mt-1">à {city}</p>
        </div>

        <button className="
          flex items-center gap-2 h-10 px-4
          bg-surface-1 border border-hairline rounded-control
          text-label text-ink
          hover:bg-surface-2 transition-colors duration-DEFAULT
        ">
          Trier par : <span className="font-semibold">Pertinence</span>
        </button>
      </div>

      {/* Carte */}
      <div className="relative w-full h-[420px] rounded-card overflow-hidden border border-hairline mb-8">
        <Map
          mapboxAccessToken={process.env.NEXT_PUBLIC_MAPBOX_TOKEN}
          initialViewState={{
            longitude: results[0]?.longitude ?? -7.9811,
            latitude: results[0]?.latitude ?? 31.6295,
            zoom: 12,
          }}
          mapStyle="mapbox://styles/mapbox/light-v11"
          style={{ width: "100%", height: "100%" }}
        >
          <NavigationControl position="top-right" />

          {results.map((salon) => (
            <Marker
              key={salon.id}
              longitude={salon.longitude}
              latitude={salon.latitude}
              onClick={() => setSelectedId(salon.id)}
            >
              <div
                className={`
                  flex items-center justify-center
                  w-9 h-9 rounded-full cursor-pointer
                  transition-colors duration-DEFAULT
                  ${selectedId === salon.id
                    ? "bg-accent text-canvas-pure"
                    : "bg-surface-1 border border-hairline-strong text-ink"}
                `}
              >
                {/* Icône salon — Lucide React, ex: Scissors */}
              </div>
            </Marker>
          ))}
        </Map>
      </div>

      {/* Grille de résultats */}
      <div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-6">
        {results.map((salon) => (
          <SalonResultCard
            key={salon.id}
            salon={salon}
            isSelected={selectedId === salon.id}
            onHover={() => setSelectedId(salon.id)}
          />
        ))}
      </div>

      {/* Pagination — explicite, pas de scroll infini */}
      <div className="flex items-center justify-center gap-2 mt-12">
        <button
          disabled={page === 1}
          onClick={() => setPage((p) => p - 1)}
          className="
            h-10 px-4 rounded-control border border-hairline
            text-label text-ink-secondary
            hover:bg-surface-2 transition-colors duration-DEFAULT
            disabled:opacity-40 disabled:cursor-not-allowed
          "
        >
          Précédent
        </button>

        {Array.from({ length: totalPages }, (_, i) => i + 1).map((p) => (
          <button
            key={p}
            onClick={() => setPage(p)}
            className={`
              w-10 h-10 rounded-control text-label
              transition-colors duration-DEFAULT
              ${p === page
                ? "bg-accent text-canvas-pure"
                : "text-ink-secondary hover:bg-surface-2"}
            `}
          >
            {p}
          </button>
        ))}

        <button
          disabled={page === totalPages}
          onClick={() => setPage((p) => p + 1)}
          className="
            h-10 px-4 rounded-control border border-hairline
            text-label text-ink-secondary
            hover:bg-surface-2 transition-colors duration-DEFAULT
            disabled:opacity-40 disabled:cursor-not-allowed
          "
        >
          Suivant
        </button>
      </div>
    </div>
  );
}
```

### Card de résultat (cohérente avec le SalonCard déjà défini dans le design system)

```tsx
function SalonResultCard({ salon, isSelected, onHover }: {
  salon: SalonResult;
  isSelected: boolean;
  onHover: () => void;
}) {
  return (
    <div
      onMouseEnter={onHover}
      className={`
        group bg-surface-1 border rounded-card overflow-hidden cursor-pointer
        transition-colors duration-DEFAULT
        ${isSelected ? "border-accent-hairline" : "border-hairline hover:border-hairline-strong"}
      `}
    >
      <div className="relative aspect-[4/3] overflow-hidden">
        <img
          src={salon.imageUrl}
          alt={salon.name}
          className="w-full h-full object-cover transition-transform duration-slow group-hover:scale-[1.03]"
        />
        {salon.badge && (
          <span className="
            absolute top-3 left-3 h-[22px] px-2
            bg-canvas-pure/90 backdrop-blur-sm
            text-caption font-medium text-ink rounded-chip
          ">
            {salon.badge}
          </span>
        )}
      </div>

      <div className="px-4 pt-3 pb-4">
        <div className="flex items-start justify-between">
          <div>
            <h3 className="text-body font-semibold text-ink">{salon.name}</h3>
            <p className="text-body-s text-ink-tertiary mt-0.5">{salon.city}</p>
          </div>
          <span className="text-body-s text-ink-tertiary">{salon.priceLevel}</span>
        </div>

        <div className="flex items-center gap-1.5 mt-2">
          <span className="text-body-s font-medium text-ink">{salon.rating}</span>
          <span className="text-body-s text-ink-tertiary">· {salon.reviewCount} avis</span>
        </div>

        <div className="border-t border-hairline my-3" />

        <p className="text-caption font-medium text-accent">
          Disponible à {salon.nextSlot}
        </p>
      </div>
    </div>
  );
}
```

## Comportement attendu (interaction carte ↔ cards)

1. Survoler une card de résultat **met en évidence le marker
   correspondant** sur la carte (couleur accent au lieu de blanc/hairline)
   — c'est le détail qui fait que la carte et la liste se sentent
   connectées plutôt que juxtaposées.
2. Cliquer un marker peut faire défiler la liste jusqu'à la card
   correspondante (`scrollIntoView`), optionnel mais recommandé si le temps
   le permet.
3. La carte ne recharge PAS sa position au changement de page de
   pagination, sauf si les résultats de la nouvelle page sont
   géographiquement très éloignés (dans ce cas, recentrer/refit les bounds
   sur les nouveaux résultats).

## Responsive (cohérent avec le système déjà en place)

- Carte : hauteur réduite sur mobile (`h-[280px] md:h-[420px]`), jamais une
  hauteur fixe identique à toutes les tailles.
- Grille de résultats : `grid-cols-1 sm:grid-cols-2 lg:grid-cols-4` —
  jamais 4 colonnes fixes sans variante mobile/tablette.
- Pagination : sur mobile, peut se réduire à juste "Précédent / Page X sur
  Y / Suivant" sans afficher tous les numéros de page individuellement si
  l'espace est insuffisant.
