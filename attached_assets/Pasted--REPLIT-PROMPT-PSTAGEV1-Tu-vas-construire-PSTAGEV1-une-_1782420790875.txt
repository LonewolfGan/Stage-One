# REPLIT PROMPT — PSTAGEV1

Tu vas construire PSTAGEV1 — une plateforme SaaS de réservation en ligne
pour salons de coiffure et instituts de beauté au Maroc.

FICHIERS DE RÉFÉRENCE — lis-les tous avant d'écrire la moindre ligne :
- CLAUDE.md        → vision produit, stack, modèle de données, règles métier
- PSTAGEV1-PRD.md  → exigences fonctionnelles complètes
- DESIGN_SYSTEM.md → tokens CSS, composants Tailwind, règles visuelles
- REPLIT_CONTEXT.md → structure des pages, UX flows, composants détaillés

## IDENTITÉ VISUELLE — le rose comme ADN

La couleur signature est #D4547A — rose profond inspiré Airbnb,
adapté à l'univers beauté premium. Planity n'a pas de couleur forte.
C'est notre avantage concurrentiel visuel.

Palette à utiliser EXCLUSIVEMENT (dans globals.css dès le départ) :

--accent:           #D4547A   → CTA primaires, éléments actifs
--accent-hover:     #C04268   → Hover boutons primaires
--accent-light:     #FAF0F3   → Backgrounds sélectionnés, tags actifs
--accent-border:    #F0C4CF   → Bordures légères en contexte accent
--canvas:           #FFFFFF   → Fond général
--canvas-subtle:    #FAFAFA   → Fonds de sections alternées
--ink:              #1A1A1A   → Texte principal
--ink-secondary:    #6B6B6B   → Texte secondaire
--ink-tertiary:     #9E9E9E   → Placeholders
--border:           #F0F0F0   → Séparateurs neutres
--rating:           #F5A623   → Étoiles uniquement
--success:          #43A047
--error:            #E53935

Règles d'usage du rose :
- Bouton primaire → fond #D4547A, texte blanc (jamais fond noir)
- Créneau/date sélectionné → fond #FAF0F3, bordure #D4547A
- Badge "Vérifié" → fond #FAF0F3, texte #D4547A
- Focus ring inputs → ring-[#D4547A]/20
- Maximum 10% de surface par écran — accent, pas masse

## RÈGLES DE DESIGN — non négociables

1. Lis DESIGN_SYSTEM.md en entier avant le premier composant.
   Utilise exactement les classes Tailwind et tokens définis dedans.
   Aucune couleur, espacement ou border-radius arbitraires.

2. Lis REPLIT_CONTEXT.md pour la structure exacte de chaque page.
   Respecte les layouts et patterns UX — mais PAS l'identité visuelle
   de Planity. On s'inspire de leur UX, on crée notre propre look.

3. Police Inter depuis Google Fonts — dans le layout global.

4. Shadcn/ui comme base. Installe au départ :
   npx shadcn@latest add button input card badge separator
   avatar tabs sheet dialog scroll-area

5. Skeleton loading sur chaque liste (shimmer #F5F5F5 → #FAF0F3).
   État vide = icône + titre + sous-titre. Jamais de page blanche.

6. Micro-animations obligatoires :
   - Card hover : translateY(-2px) + shadow elevation
   - Button active : scale(0.97)
   - Slot sélectionné : transition 150ms avec fond #FAF0F3

## PARTIS PRIS UX — innover par rapport à Planity

1. SALONCARD — afficher "Disponible à 14h30" en rose en évidence,
   pas une grille de slots anonymes. Le client voit la prochaine
   dispo immédiatement.

2. PAGE FICHE — badge "Populaire" en rose sur le service le plus
   réservé. Layout photo asymétrique (hero large + vignettes).

3. BOOKING WIDGET — deux colonnes desktop : sélection à gauche,
   récapitulatif live à droite. Bottom sheet sur mobile.
   Pas de stepper numéroté visible.

4. DASHBOARD — style Linear : sidebar #FAFAFA, item actif en rose,
   bordure gauche rose sur les blocs de RDV. Hero cards pour
   les 3 prochains RDV en haut de l'agenda.

5. EMPTY STATES — micro-illustrations SVG inline avec rose comme
   couleur d'accent.

Ce produit doit ressembler à ce qu'Airbnb aurait construit
pour la réservation beauté au Maroc.
Référence UX : Planity. Référence dashboard : Linear.
Identité visuelle : la nôtre.
