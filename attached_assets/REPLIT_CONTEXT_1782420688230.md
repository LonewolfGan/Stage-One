# REPLIT_CONTEXT.md — Booking Platform UX Architecture

> **Research Source:** Planity (planity.com) — French beauty booking platform  
> **Analysis Date:** 2026-06-25  
> **Design System:** Airbnb-inspired (pure white, deep ink, Inter, soft radii)  

---

## 1. Design System Tokens

### Color Palette
| Token | Value | Usage |
|-------|-------|-------|
| `--canvas` | `#FFFFFF` | Page background, card surfaces |
| `--ink` | `#1A1A1A` | Primary text, headings |
| `--ink-secondary` | `#6B6B6B` | Secondary text, descriptions |
| `--ink-tertiary` | `#9E9E9E` | Placeholder, disabled states |
| `--border` | `#F0F0F0` | Dividers, card borders |
| `--accent` | `#1A1A1A` | CTA buttons, active states |
| `--accent-hover` | `#333333` | CTA hover state |
| `--rating` | `#F5A623` | Star ratings |
| `--tag-bg` | `#F5F5F5` | Category/tag badges |
| `--error` | `#E53935` | Error states |
| `--success` | `#43A047` | Confirmation states |

### Typography
- **Font:** Inter (sans-serif), all weights
- **Weight scale:** 400 (regular), 500 (medium), 600 (semibold)
- **Size scale:** 12px (caption), 14px (body), 16px (subheading), 20px (heading), 28px (display)
- **Line height:** 1.4 (headings), 1.5 (body)

### Border Radii
- **Buttons:** 8px
- **Cards:** 14px
- **Avatars:** 50% (circular)
- **Inputs:** 8px
- **Modals:** 14px
- **Tags/Badges:** 8px

### Spacing Grid
Base unit: 4px  
`4-8-12-16-20-24-32-40-48-64-80`

### Shadow System
- **Card shadow:** `0 1px 3px rgba(0,0,0,0.06), 0 1px 2px rgba(0,0,0,0.04)`
- **Elevated:** `0 4px 12px rgba(0,0,0,0.08)`
- **Modal:** `0 20px 60px rgba(0,0,0,0.15)`

### Image Placeholders
- **Aspect ratio:** 1:1 (strict)
- **Resolution:** 1024×1024 px (simulated)
- **Object-fit:** cover
- **Border-radius:** 14px (gallery cards), 8px (service icons)
- **Fallback:** Gradient placeholder (#F5F5F5 → #E8E8E8) with centered icon

#### Key Insights
- **No page navigation** — the booking flow is a single-page widget/modal
- **Steps are shown as visual stepper**, but the step 3 summary only renders after time selection
- **15-minute slot increments** — granular enough for appointment-based services
- **Dates extend ~5 months out** with horizontal scroll navigation
- **Staff selection is a simple chip group** with initials (circular avatars, 32px)
- **Closed days shown as dimmed** (no clickable slots) — Sundays and Mondays off for this salon
- **"Add another service"** allows multi-service booking without leaving the flow
- **Arrow navigation** at the bottom scrolls time slots by week chunks

## 3. User Flows

### 3.1 Primary Flow: Book an Appointment

```
Homepage → Search → Results → Provider Page → Booking Widget → Confirmation

1. Homepage
   ├── User enters category + location in SearchPill
   └── OR clicks a category card

2. Search Results
   ├── Browse salon cards with inline availability
   ├── Filter by date (chip selector)
   ├── Click time slot → navigates to provider page (pre-selected)
   └── OR click salon card → navigates to provider public page

3. Provider Public Page
   ├── Browse photo gallery (trust signal)
   ├── Browse service categories
   ├── Click "Choisir" on a service
   └── OR "Réserver" in sticky bottom bar

4. Booking Widget (reservation page)
   ├── Step 1: Review selected service, choose staff member
   │   └── Optional: "Ajouter une prestation" for multi-service
   ├── Step 2: Select date + time slot from grid
   ├── Step 3: Review summary → Confirm
   └── Authentication gate (login/guest) before final confirmation
```

### 3.2 Secondary Flow: Quick Book from Search

```
Search Results
│
├── Select date chip
├── See available time slots on card
├── Click a time slot → direct to reservation page (service pre-selected?)
└── Faster path, skips provider page
```

### 3.3 Mobile Considerations
- **Bottom sheet** pattern for booking widget (slides up from bottom)
- **Sticky CTA** on provider page (floating action bar)
- **Category nav** collapses to hamburger menu
- **Map panel** becomes full-screen modal triggered by button
- **Time slot grid** scrolls horizontally, vertically on smaller screens

## 5. UX & Security Boundaries

### Settings / Profile Pages
- **Clean, business-focused UI** — no internal architecture exposed
- Profile fields: name, email, phone, preferred salons, booking history
- Payment methods shown as saved cards (last 4 digits, brand logo) — no full PAN
- **Never display:** JWT tokens, API keys, bcrypt references, biometric schemas, session identifiers
- Authentication flows use a generic "Se connecter" / "Mon compte" pattern — no technical mechanism shown
- Password change: standard form with current + new password fields — no hashing hints
- Error messages for auth failures are generic ("Email ou mot de passe incorrect") — never hint at which field failed

### Booking Security Boundaries
- No pricing manipulation: prices are server-validated, not derived from client-side parameters
- Slot availability: confirmed at submission, not just rendered — stale slots show "indisponible"
- Staff selection: verified against staff-salon association on the server
- Guest booking: limited to session-based identity (email + phone verification)

## 8. Error & Empty States

### Time Slot Unavailable
- **Day-level:** Date card dimmed, no clickable slots — shows day name and date only
- **Time-level:** Slot button missing from grid (not disabled, just absent)
- **Reason:** Closed day (Sunday/Monday) or fully booked

### Service Unavailable
- Service card exists in list but "Choisir" button shows "Indisponible" or hidden
- Price range may show "—" (no data)

### Photo Gallery Empty
- Placeholder: gradient background with camera icon
- Alternative: salon logo as fallback

### Search No Results
- "Aucun salon trouvé" with suggestion chip to broaden search
- Map shows the searched area center with a pin
- Category filter resets offered as quick action

### Booking is Unsuccessful
- Slot already taken: "Ce créneau n'est plus disponible" with suggestions for next available
- Salon closes: booking attempt outside hours → "Le salon est fermé à cette heure"
- Authentication error: generic "Veuillez réessayer" message

## 9. Visual & Animation Guidelines

### Micro-interactions
- **Card hover:** Slight translateY(-2px) + shadow elevation
- **Button hover:** Background darkens (accent → accent-hover)
- **Time slot tap:** Scale(0.97) flash on press, then selection state
- **Staff select:** Smooth radio-button-like transition with checkmark
- **Step transition:** Content fades/slides within the widget (no page reload)

### Loading States
- **Skeleton loading** for salon cards (shimmer animation)
- **Spinner** for time slot loading (centered, accent color)
- **Skeleton text lines** for service descriptions
- Image placeholders use background gradient shimmer until loaded

### Success States
- **Confirmation modal:** Checkmark animation + summary card
- **Toast notification:** "Rendez-vous confirmé" with undo/dismiss action
- **Email confirmation trigger** (no email UI shown — just "Un email de confirmation vous a été envoyé")

## 10. Performance Considerations
- **Image lazy loading** for gallery photos (below-fold)
- **Pre-fetch** nearby dates' time slots when hovering on a date chip
- **Virtualized time slot grid** for large date ranges (5+ months of 15-min slots)
- **Service categories** loaded eagerly (always visible)
- **Static map tiles** cached (Mapbox or similar)
- **Critical path:** SearchPill → Results → Time slot data (pre-fetched on results hover)
