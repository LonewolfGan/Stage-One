# INSTRUCTION DE DESIGN SYSTEM — PSTAGEV1

## Le problème qu'on corrige

Jusqu'ici, le design a été donné composant par composant avec des classes
Tailwind écrites en dur dans des exemples JSX (`bg-[#D4547A]`,
`rounded-[14px]`, etc.). Résultat : plusieurs versions de couleurs/radius
légèrement différentes ont été mélangées dans le code généré (un hero en
dark mode alors que le reste est en light mode, des radius trop arrondis,
des shadows qui ne devraient pas exister). **Ce prompt change la méthode,
pas seulement le contenu.**

## La nouvelle règle, à appliquer immédiatement et de façon permanente

**`tailwind.config.ts` (fourni ci-dessous, à copier-coller tel quel à la
racine de `apps/web`) est la SEULE source de vérité du design.** Il n'y a
plus aucune autre référence de design system à consulter. Si une ancienne
version de `DESIGN_SYSTEM.md` existe dans le repo, **supprime-la ou
renomme-la en `DESIGN_SYSTEM.OLD.md`** pour qu'elle ne soit plus jamais
lue par erreur.

### Ce que ça veut dire concrètement pour chaque ligne de JSX que tu écris

1. **Aucune couleur hexadécimale n'apparaît jamais dans un fichier `.tsx`.**
   Pas de `bg-[#FFFFFF]`, pas de `text-[#1A1A1A]`, pas de `border-[rgba(0,0,0,0.1)]`.
   Tu utilises exclusivement les noms définis dans `tailwind.config.ts` :
   `bg-canvas`, `bg-surface-1`, `bg-surface-2`, `bg-surface-3`, `text-ink`,
   `text-ink-secondary`, `text-ink-tertiary`, `bg-accent`, `text-accent`,
   `border-hairline`, `border-hairline-strong`, etc.
2. **Aucune taille de police arbitraire.** Pas de `text-[15px]`, pas de
   `text-2xl`, pas de `text-lg`. Tu utilises exclusivement :
   `text-display-hero`, `text-display`, `text-heading-l`, `text-heading-m`,
   `text-heading-s`, `text-body-l`, `text-body`, `text-body-s`, `text-label`,
   `text-caption`. Chacune de ces classes embarque déjà le bon poids, le bon
   line-height et le bon tracking — tu n'ajoutes jamais `font-bold`,
   `leading-*` ou `tracking-*` à côté, ce serait redondant ou contradictoire.
3. **Aucun radius arbitraire.** Pas de `rounded-[10px]`, pas de
   `rounded-2xl`. Tu utilises exclusivement : `rounded-chip` (4px),
   `rounded-tag` (6px), `rounded-control` (8px, boutons/inputs),
   `rounded-card` (12px, cards/modals), `rounded-panel` (16px),
   `rounded-full` (pills/avatars).
4. **Aucun espacement arbitraire.** Pas de `p-[13px]`, pas de `gap-[19px]`.
   Tu utilises exclusivement les valeurs de l'échelle `spacing` du config
   (1, 2, 3, 4, 5, 6, 8, 12, 16, 24, 32 → 4px à 128px).
5. **Aucun `box-shadow`, jamais, nulle part.** Aucune classe `shadow-*`
   n'existe dans ce projet — regarde `tailwind.config.ts`, le bloc
   `boxShadow` est vide intentionnellement. Si tu penses qu'un composant a
   besoin d'une ombre pour se détacher du fond, c'est que tu dois plutôt
   monter d'un niveau dans la surface ladder (`bg-surface-1` →
   `bg-surface-2` → `bg-surface-3`) et/ou ajouter une `border-hairline`.
6. **Container de page** : jamais `max-w-6xl`, `max-w-7xl`, ou tout
   container Tailwind par défaut. Utilise `max-w-page` (1680px) défini
   dans le config, avec un padding horizontal responsive
   (`px-5 md:px-10 lg:px-20`).
7. **Si une valeur dont tu as besoin n'existe pas dans
   `tailwind.config.ts`** (une couleur, une taille, un radius), **tu ne
   l'inventes pas**. Tu t'arrêtes et tu me demandes d'étendre le fichier de
   config. Ajouter une valeur hors de ce fichier — même "juste cette
   fois", même "ça ne se voit presque pas" — recrée exactement le problème
   qu'on corrige.

### Vérification mécanique obligatoire après chaque lot de composants

Avant de me dire qu'un composant ou une page est terminé, exécute ces
commandes et montre-moi leur sortie réelle (pas un résumé verbal) :

```bash
# Doit retourner 0 résultat — aucune couleur hex en dur
grep -rEn "#[0-9A-Fa-f]{3,6}" --include="*.tsx" apps/web/app apps/web/components

# Doit retourner 0 résultat — aucune classe arbitraire de couleur/taille/radius/spacing
grep -rEn "(bg|text|border)-\[#|text-\[[0-9]+px\]|rounded-\[[0-9]+px\]|p-\[[0-9]+px\]|gap-\[[0-9]+px\]" --include="*.tsx" apps/web/app apps/web/components

# Doit retourner 0 résultat — aucun shadow
grep -rn "shadow-" --include="*.tsx" apps/web/app apps/web/components

# Doit retourner 0 résultat — aucun container générique Tailwind
grep -rEn "max-w-(6xl|7xl|5xl|4xl)\b" --include="*.tsx" apps/web/app apps/web/components
```

Si une de ces commandes retourne un résultat non vide, le composant n'est
**pas terminé** — corrige-le pour qu'il utilise les noms sémantiques du
config, puis relance la vérification.

## Exemples d'usage correct (référence — pas à copier littéralement, juste pour montrer le pattern de nommage attendu)

```tsx
// Bouton primaire
<button className="
  inline-flex items-center justify-center gap-2 h-11 px-5
  bg-accent text-canvas-pure text-label
  rounded-control transition-colors duration-DEFAULT
  hover:bg-accent-hover
">

// Card
<div className="
  bg-surface-1 border border-hairline rounded-card overflow-hidden
  transition-colors duration-DEFAULT
  hover:bg-surface-2 hover:border-hairline-strong
">

// Titre de section
<h2 className="text-display text-ink">Explorez par catégorie</h2>

// Container de page
<main className="max-w-page mx-auto px-5 md:px-10 lg:px-20">
```

Remarque comment aucune de ces lignes ne contient de hex, de px arbitraire,
ou de shadow. Si tu te retrouves à écrire un crochet `[...]` dans une classe
Tailwind pour une couleur, une taille de texte, un radius ou un espacement,
arrête-toi — c'est le signal que tu es en train de sortir du design system.

## Ce qu'il faut faire avec le code déjà généré (le hero noir, etc.)

Avant d'ajouter quoi que ce soit de nouveau :
1. Installe le nouveau `tailwind.config.ts` ci-dessus.
2. Repasse sur les fichiers déjà écrits (hero, cards, dashboard) et
   remplace chaque valeur en dur par le nom sémantique correspondant le
   plus proche dans le config. Si une couleur existante (ex: le fond noir
   du hero) n'a pas d'équivalent dans le config, c'est volontaire — elle
   n'est plus autorisée, remplace-la par `bg-canvas` ou `bg-surface-2`
   selon le contexte, pas par une approximation hexadécimale.
3. Lance les 4 commandes `grep` de vérification ci-dessus sur l'ensemble
   du dossier `apps/web` et montre-moi le résultat avant de continuer sur
   quoi que ce soit d'autre.
