# DESIGN_SYSTEM.md — Booking Platform Visual Identity

> **RÈGLE ABSOLUE POUR REPLIT / CLAUDE CODE**  
> Ce fichier est une contrainte de développement, pas une suggestion.  
> Chaque composant React produit DOIT respecter ce système à la lettre.  
> Un composant fonctionnel avec un mauvais design est un composant non livré.  
> Référence visuelle : Airbnb, Linear, Vercel, Planity — niveau Awwwards.

## 5. Composants spécifiques au produit

### SalonCard (Search Results)

```
┌──────────────────────────────────────┐
│  Image 1:1 (aspect-square)           │  ← rounded-[14px], object-cover
│  → swipeable carousel                │
│  → Badge flottant top-left            │
├──────────────────────────────────────┤
│  Nom salon        16px semibold       │  ← mt-3
│  Adresse          14px secondary      │  ← mt-1
│  ★ 4.8 · 97 avis  14px · €€€         │  ← mt-1 flex items-center gap-2
├──────────────────────────────────────┤  ← separator mt-3 mb-3
│  [26 juin] [27 juin] [28 juin]        │  ← date chips scrollable
│  [10:00·49€] [10:30·49€] [+2]         │  ← slot pills mt-2
└──────────────────────────────────────┘
```

Règles :
- Image : `aspect-square w-full object-cover rounded-[14px]`
- Slot pills : `h-9 px-3 border border-[#E0E0E0] rounded-[8px] text-[13px] font-medium hover:border-[#1A1A1A]`
- Date chips : `h-7 px-3 bg-[#F5F5F5] rounded-full text-[12px] font-medium`
- Card padding : `p-0` (image full-bleed) + `px-1 pb-3` pour le texte sous l'image

### ServiceCard (Provider Page)

```
┌──────────────────────────────────────┐
│  Nom du service        16px semibold  │
```

## 8. Images — règles strictes

- Toujours `object-cover` — jamais `object-contain` sur les photos de salons
- Ratio **1:1** pour les cartes de résultats et profils staff
- Ratio **16:9** ou **3:2** uniquement pour les hero banners
- **Placeholder** (pendant le chargement) :
  ```tsx
  <div className="bg-gradient-to-br from-[#F5F5F5] to-[#E8E8E8] animate-pulse" />
  ```
- **Fallback** (si image absente) :
  ```tsx
  <div className="bg-[#F5F5F5] flex items-center justify-center">
    <ScissorsIcon className="w-8 h-8 text-[#C4C4C4]" />
  </div>
  ```

## 9. États vides — jamais laisser une page blanche

Chaque état vide doit avoir :
1. Une illustration SVG inline simple (pas d'image externe)
2. Un titre court (max 5 mots)
3. Un sous-titre explicatif (max 15 mots)
4. Un CTA primaire si applicable

## 10. Règles absolues — ne jamais violer

1. **Pas de couleurs arbitraires** — uniquement les tokens définis en section 1
2. **Pas de `margin` ou `padding` aléatoires** — uniquement les multiples de 4px
3. **Pas de `font-size` en dehors de** : 12, 13, 14, 15, 16, 18, 20, 24, 28, 32, 40px
4. **Pas de `border-radius` en dehors de** : 6, 8, 14, 20, 9999px
5. **Pas de page sans état de chargement** — skeleton obligatoire sur chaque liste
6. **Pas d'état vide blanc** — voir section 9
7. **Pas de `!important`** dans le CSS
8. **Pas de styles inline** sauf exceptions documentées
9. **Toujours `transition`** sur les éléments interactifs (hover, focus, active)
10. **Shadcn/ui comme base** — personnaliser par-dessus, jamais recréer from scratch
