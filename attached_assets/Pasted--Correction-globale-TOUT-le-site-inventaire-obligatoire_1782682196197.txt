# Correction globale — TOUT le site, inventaire obligatoire avant correction

## Pourquoi ce prompt est structuré comme ça

La dernière fois, seule la hero section a été modifiée alors que la
demande concernait tout le site. Cause probable : sans inventaire
explicite, seul ce qui est visible par défaut dans le flux de navigation
est traité — un modal qui ne s'ouvre que sur clic (ex: "Commencer" dans le
header) n'apparaît jamais dans un audit visuel classique. **Ce prompt
commence donc par un inventaire par le CODE, pas par la navigation, pour
forcer à traiter des éléments qu'on ne voit jamais sans interaction.**

---

## ÉTAPE 3 — Règles mécaniques à appliquer fichier par fichier

Pour CHAQUE fichier listé à l'étape 1, applique ces règles. Pas de
jugement contextuel à inventer — ce sont des règles fixes :

### Boutons — hiérarchie rose/noir
- Bouton qui déclenche l'action principale d'un écran (valider, créer,
  réserver, rechercher, commencer, confirmer) → **fond rose, texte blanc**.
- Bouton secondaire à côté (annuler, retour, plus tard, fermer) → **fond
  noir ou outline noir, texte noir/blanc selon le variant**, jamais rose.
- Un seul bouton rose par groupe d'actions visible simultanément. S'il y a
  3 boutons côte à côte, un seul est rose (le principal), les autres sont
  noirs/outline.

### Border-radius — cohérence conteneur/enfant
**Règle mécanique simple : le radius d'un élément enfant ne dépasse jamais
le "type" de radius de son conteneur direct.**
- Si un conteneur (ex: search bar, card de filtre) a un radius non-pill
  (`rounded-control` 8px ou `rounded-card` 12px), tout bouton/input à
  l'intérieur de ce conteneur doit aussi être en radius non-pill — jamais
  `rounded-full` à l'intérieur d'un conteneur à coins carrés/légèrement
  arrondis.
- Si un conteneur est en `rounded-full` (pill), les éléments à l'intérieur
  peuvent être en pill aussi.
- Vérifie ça explicitement sur : la search bar du hero, les filtres de la
  page résultats, tout composant avec un input + bouton accolés.

### Texte gris — interdiction totale, sans exception de fichier
```bash
grep -rn "text-gray-\|#8C8A82\|#54534D\|#6B6B6B\|#9E9E9E" --include="*.tsx" apps/web
```
Doit retourner 0 résultat sur l'ENSEMBLE des fichiers de l'étape 1, modals
et dropdowns inclus. Remplace par `text-ink-secondary`
(`rgba(12,12,14,0.65)`) ou `text-ink-tertiary` (`rgba(12,12,14,0.45)`).

### Espacement — aération systématique
Le site doit respirer, pas être dense/confus. Règles fixes :
- Padding interne minimum d'une card/panel : `p-6` (24px), jamais `p-2`/
  `p-3` sur un contenant principal.
- Gap minimum entre éléments d'une liste/grille : `gap-4` (16px) en
  dessous, `gap-6` (24px) au-dessus de 3 colonnes.
- Marge verticale entre sections d'une page : jamais moins de `py-12`
  (48px), `py-16`/`py-24` pour les sections majeures de la home.
- Dans un modal/dialog : padding interne `p-6` minimum, espace `mt-4`
  minimum entre le titre et le corps, entre le corps et les boutons
  d'action.
- Si un élément actuel a un padding/gap inférieur à ces minimums,
  augmente-le. Ne réduis jamais en dessous de ces valeurs.
