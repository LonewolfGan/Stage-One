# Rééquilibrage palette + utilisation OBLIGATOIRE du skill Impeccable

## ⚠️ Étape 0 — à exécuter et VÉRIFIER avant toute autre action

La dernière fois, l'installation du skill Impeccable demandée n'a pas été
faite, et le travail a continué sans lui. **Cette fois, ne fais RIEN
d'autre avant d'avoir confirmé chacune de ces 3 étapes avec une preuve
concrète (pas une affirmation verbale) :**

1. Exécute :
   ```bash
   npx skills add https://github.com/pbakaus/impeccable --skill impeccable
   ```
   Montre la sortie réelle de la commande.

2. Vérifie que le skill est bien installé en listant le dossier où il
   devrait se trouver (`.agents/skills/`, `~/.agents/skills/`, ou
   `.github/skills/` selon l'outil) :
   ```bash
   find . -iname "*impeccable*" -not -path "*/node_modules/*"
   ```
   Montre la sortie réelle. Si elle est vide, l'installation a échoué —
   recommence l'étape 1 avant de continuer, ne passe pas à l'étape 3 tant
   que ce dossier n'existe pas.

3. Lance `/impeccable init` et confirme que `PRODUCT.md` (et `DESIGN.md`)
   ont été écrits à la racine du projet :
   ```bash
   cat PRODUCT.md
   ```
   Si ce fichier n'existe pas, l'init a échoué — recommence avant de
   continuer.

**Ne passe à la section suivante que si les 3 preuves ci-dessus ont été
montrées et sont positives.** Si à un moment le skill n'est pas
disponible dans cet environnement (Replit ne le supporte pas), dis-le
explicitement plutôt que de continuer sans lui silencieusement.

## Les 3 corrections demandées — règles précises

### 1. Border-radius en pill — à RÉTABLIR

Les boutons (CTA primaires/secondaires), les chips de catégories, les tags
de filtre doivent revenir à `rounded-full` (pill), comme c'était avant le
dernier passage. Seuls les composants suivants restent en `rounded-card`
(12px), pas en pill : cards de résultats, modals, panels de contenu, inputs
de formulaire classiques (pas les boutons).

### 2. Rose — à rééquilibrer, PAS sur le texte

- Le rose réapparaît sur : les boutons (alterner entre boutons noirs et
  boutons roses selon le contexte — par exemple le CTA le plus important
  d'un écran peut être rose, les actions secondaires restent noires ou
  outline), les badges d'état actif, les éléments d'accent visuel
  (indicateur de sélection, focus ring, icône active).
- Le rose n'apparaît JAMAIS sur du texte (pas de mot coloré dans un titre,
  pas de lien rose, pas de prix en rose). Le texte reste toujours en
  `ink` (noir) ou `ink-secondary` (noir à opacité réduite, voir point 3).
- Le hero de la page d'accueil doit avoir une présence rose visible (sur
  son CTA principal par exemple), pas seulement le reste du site.
- Pas de règle stricte de "1 seul élément rose par écran" cette fois — le
  rose peut apparaître sur plusieurs éléments (plusieurs boutons, badges)
  dans un même écran, à condition de rester sur des éléments d'interface
  (boutons/badges/accents), jamais sur du texte courant.

### 3. Texte gris — à éliminer complètement

```ts
// Dans tailwind.config.ts
"ink-secondary": "rgba(12, 12, 14, 0.65)",
"ink-tertiary": "rgba(12, 12, 14, 0.45)",
```

Toute classe de texte qui utilise actuellement une couleur grise séparée
(`text-gray-400`, `#8C8A82`, ou toute autre valeur grise/beige) doit être
remplacée par `text-ink-secondary` ou `text-ink-tertiary` ci-dessus — donc
une opacité réduite du même noir de marque, jamais une teinte grise
indépendante.

## Exécution avec Impeccable — workflow complet

Une fois l'étape 0 confirmée et `init` fait avec le contexte ci-dessus :

```
/impeccable audit la page d'accueil et le hero
/impeccable audit la page de recherche et résultats
/impeccable audit la fiche prestataire et le booking
/impeccable audit l'onboarding et les écrans d'authentification
/impeccable audit le dashboard prestataire
```

Pour chaque audit, demande explicitement de vérifier :
- Présence de border-radius en pill sur les boutons/chips (doit être
  présent, pas absent — c'est l'inverse du dernier audit).
- Présence du rose sur au moins les CTA principaux, en particulier le hero.
- Absence de toute couleur grise/beige en dehors de `ink-secondary`/
  `ink-tertiary` définis ci-dessus.

Puis corrige avec :
```
/impeccable polish la page d'accueil et le hero
/impeccable polish la page de recherche et résultats
/impeccable polish la fiche prestataire et le booking
/impeccable polish l'onboarding
/impeccable polish le dashboard prestataire
```

Active le hook de surveillance pour éviter une nouvelle régression
silencieuse :
```
/impeccable hooks on
```
