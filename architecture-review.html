Centralize the error response interface

Backend · routes/*.ts (19 files with inline error codes)

Strong in-process

Before — hardcoded error codes everywhere

bookings.ts: res.status(404).json({code:"ERR-004"})
providers.ts: res.status(404).json({code:"ERR-004"})
dashboard.ts: res.status(403).json({code:"ERR-003"})
16 more files with the same inline pattern…

After — typed codes, one interface

lib/errors.ts

notFound(res, msg?)

forbidden(res, msg?)

conflict(res, "ERR-005", msg)

badRequest(res, errors)

Problem

Error codes ("ERR-004", "ERR-003"…) are inline strings in 19 route files. A typo changes the contract silently. Changing the shape requires touching every file.

Solution

Typed AppErrorCode union + convenience helpers in lib/errors.ts. Routes call notFound(res) not res.status(404).json({…}).

Wins

  • ✓ leverage: one interface, 19+ call sites
  • ✓ TypeScript enforces valid error codes
  • ✓ locality: change error shape in one module

✓ Implemented — lib/errors.ts created. Migrating individual routes is an additive follow-up.

Stop localStorage leaking through the API module interface

Frontend · lib/api.ts · App.tsx → owned by lib/auth-store.ts

Strong in-process

Before — storage key is a hidden shared dependency

api.ts: localStorage.getItem("auth_token")
App.tsx: localStorage.setItem("auth_token", d.token)

"auth_token" key is an implicit contract between 3 files. DevAutoLogin also missed setting the refresh token and user object.

After — storage owned behind auth-store interface

lib/auth-store.ts

getToken() · setTokens() · clearTokens()

api.ts → getToken() App.tsx → setTokens()

Problem

Raw localStorage.getItem("auth_token") in api.ts — auth-store already owns the key via getToken() but it wasn't used. DevAutoLogin also set only the token, missing refreshToken and user.

Solution

api.ts calls getToken() from auth-store. App.tsx calls setTokens(token, refresh, user) — now correctly stores all three fields.

Wins

  • ✓ locality: storage key owned by one module
  • ✓ DevAutoLogin now stores full auth state
  • ✓ swap to sessionStorage in one file
  • ✓ deleting getItem("auth_token") from api.ts passes deletion test

✓ Implemented — api.ts and App.tsx now use getToken() / setTokens() from auth-store.

Problem

getAvailableSlots() runs 4–5 DB queries internally. The slot calculation logic is impossible to unit test without a running database.

Solution

Split into fetchSlotContext() (DB adapter) and computeSlots(ctx) (pure function). Two adapters — real DB + in-memory — justify the seam.

Wins

  • ✓ interface shrinks (pure fn testable in isolation)
  • ✓ seam justified: in-memory adapter for tests
  • ✓ locality: slot calculation bugs visible in one module
